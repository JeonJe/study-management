# LOOPERS MEETUP 현재 제품 상황

작성일: 2026-05-03

## 1. 제품 요약

LOOPERS MEETUP은 기수별 루프팩, 스터디, 뒷풀이, 엔젤 주간보고, 관리자 운영을 한 곳에서 처리하는 내부 운영 대시보드다.

첫 화면은 참여할 기수를 카드로 고르는 구조이며, 기수 입장 코드를 통과하면 해당 기수의 루프팩/스터디/뒷풀이 화면을 사용할 수 있다. 엔젤 화면과 관리자 화면은 같은 기수 안에서도 별도 코드로 한 번 더 확인한다. 전체 관리자는 기수 생성/편집과 코드 관리만 담당하며 기수 관리자 화면과 분리된다.

## 2. 주요 사용자

- 참여자/운영 보조: 기수 입장 코드로 루프팩, 스터디, 뒷풀이 현황을 확인한다.
- 엔젤: 담당 팀 주간보고를 작성하고 수정한다.
- 기수 관리자: 팀/멤버/엔젤 배정, 주간보고 주차/템플릿, 히스토리를 관리한다.
- 전체 관리자: 기수 목록, 접속 주소, 입장 코드, 엔젤 코드, 관리자 코드를 관리한다.

## 3. 핵심 기능

### 기수와 권한

- `/`에서 기수를 선택하고 입장 코드로 진입한다.
- `/cohorts/{unit}/entry`는 기수 입장 코드 전용 화면이다.
- `/cohorts/{unit}/angel`은 엔젤 코드, `/cohorts/{unit}/admin`은 관리자 코드가 필요하다.
- `/admin`은 전체 관리자 전용이며 기수 관리 화면에 의존하지 않는다.
- 로그아웃 후에는 `auth=required` 같은 상태 쿼리를 제거하고 항상 같은 첫 화면으로 돌아간다.

### 루프팩/스터디

- 날짜별 모임 목록과 상세를 관리한다.
- 모임별 정원은 엔젤/운영진을 포함한 전체 확정 인원 기준이다.
- 남은 자리는 확정, 초과 인원은 대기 인원으로 분리한다.
- 대기 인원은 확정 또는 취소할 수 있고, 확정 인원은 대기로 전환하거나 참여 제외할 수 있다.
- 상세 화면에서 공유 문구를 복사하고, 장소 링크는 네이버맵/카카오맵 폴백 버튼을 제공한다.

### 뒷풀이/정산

- 날짜별 뒷풀이와 정산 상태를 확인한다.
- 루프팩/스터디와 같은 상단 요약 패턴을 사용한다.
- 참여자, 정산 묶음, 정산 완료 상태를 관리한다.

### 멤버/팀 관리

- 팀은 카드 클릭 후 상세 모달에서 수정/삭제한다.
- 팀 멤버와 팀 담당 엔젤은 팀 수정 모달에서 관리한다.
- 운영 역할 디렉터리는 역할 카드 클릭 후 명단을 추가/수정/삭제한다.
- 팀에 배정된 엔젤은 팀 데이터와 충돌하지 않도록 팀 수정에서 관리한다.
- 저장 결과는 하단 중앙 토스트로 표시한다.

### 엔젤 주간보고

- 관리자는 보고 템플릿과 보고 주차를 만든다.
- 엔젤은 담당 팀 보고를 작성/수정한다.
- 관리자는 제출 현황과 내용을 확인하고 필요 시 미제출로 되돌릴 수 있다.
- 슬랙 공유 문구와 댓글 기능은 현재 범위에서 제외했다.

## 4. UI/UX 원칙

현재 화면 개선 기준은 `docs/ui-ux-principles.md`를 따른다.

- 사용자에게 “운영 단위” 같은 내부 용어를 노출하지 않는다.
- 카드에 이미 맥락이 있으면 버튼은 `추가`, `수정`, `삭제`, `확정`, `취소`처럼 행동만 적는다.
- 같은 정보를 카드/뱃지/요약에 중복해서 표시하지 않는다.
- 지속 배지 대신 하단 중앙 토스트로 결과를 알린다.
- 카드 안에 카드를 반복하지 않고, 리포트/템플릿은 구분선과 타이포그래피로 읽히게 만든다.

## 5. 현재 리스크

- 기수 코드의 현재값 표시를 위해 reversible storage가 필요하다. 새로 저장되는 코드는 앱 비밀값으로 암호화하지만, 과거 평문 저장값은 재저장 전까지 남아 있을 수 있다.
- 서버 액션과 화면이 계속 커지고 있어 다음 대형 기능 전에는 모임/뒷풀이/주간보고 UI 컴포넌트 경계를 더 나눌 필요가 있다.
- E2E는 로컬 기준으로 설계되어 있으며, 운영 유사 환경에서 실행할 때는 테스트 데이터 오염을 별도로 통제해야 한다.
