# Critical Path Checklist

신규 운영 단위를 프로덕션에 바로 여는 상황에서 핵심 흐름이 끊기지 않는지 확인하는 체크리스트입니다. 자동 검증은 `e2e/regression-critical-path.spec.ts`의 `CP-*` 단계와 1:1로 대응합니다.

이 시나리오는 실제 DB에 새 운영 단위를 만들고 삭제하는 상태 누적형 쓰기 테스트라 기본 비활성화되어 있습니다. 수동 점검이 필요할 때만 `RUN_CRITICAL_PATH_E2E=1`을 지정해 단독 실행합니다.

## 실행 원칙

- 새 운영 단위 `e2e-critical-*`를 매 실행마다 생성한다.
- 기존 운영 데이터나 `loop-pak-3` 데이터에 의존하지 않는다.
- 상태 누적형 시나리오이므로 병렬 실행과 재시도는 금지한다.
- 실패 시 같은 운영 단위 안의 어느 단계에서 터졌는지 확인하고, 무조건 재시도하지 않는다.
- 종료 시 테스트 운영 단위를 삭제해서 활성 테스트 데이터를 남기지 않는다.
- 검증은 브라우저 화면 기준으로 수행한다. E2E 내부에서 DB를 직접 조회하지 않는다.

## 체크리스트

| ID | 기능 | 검증 기준 | 자동 검증 단계 |
| --- | --- | --- | --- |
| CP-01 | 신규 운영 단위 생성 | 전체 관리자에서 slug, 이름, 입장 코드, 엔젤 코드, 관리자 코드를 입력해 새 운영 단위를 만들고 목록에 보인다. | `test.step("CP-01 ...")` |
| CP-02 | 운영 단위 관리자 로그인 | 새 운영 단위의 입장 코드와 관리자 코드로 `/cohorts/{slug}/admin`에 들어간다. 기존 세션이나 데이터에 기대지 않는다. | `test.step("CP-02 ...")` |
| CP-03 | 운영진 명단 | 팀이 하나도 없는 상태에서 엔젤, 멘토, 매니저, 서포터, 버디를 저장한다. 엔젤 수정과 임시 매니저 삭제까지 확인한다. | `test.step("CP-03 ...")` |
| CP-04 | 팀 명단 | 팀을 2개 추가하고, 주력 팀은 팀명과 멤버를 수정한다. 보조 팀은 삭제되어 목록에서 사라진다. | `test.step("CP-04 ...")` |
| CP-05 | 엔젤 주간보고 | 관리자에서 주간보고 주차를 만들고, 엔젤 화면에서 팀 보고를 작성한 뒤 수정한다. 관리자 상세에서 수정 내용이 보이고, 미제출 처리 후 작성 상태로 돌아간다. | `test.step("CP-05 ...")` |
| CP-06 | 스터디 | 스터디 모임을 만들고 팀 멤버를 빠른추가로 참여시킨다. 참여자를 제외하면 빠른추가에서 다시 추가 가능 상태가 되고, 재등록 후 모임 제목 수정까지 반영된다. | `test.step("CP-06 ...")` |
| CP-07 | 루프팩 | 루프팩 모임을 만들고 다른 팀 멤버를 빠른추가로 참여시킨다. 참여자 수와 빠른추가 상태가 즉시 반영된다. | `test.step("CP-07 ...")` |
| CP-08 | 뒷풀이 | 뒷풀이를 만들고 참여자를 추가한다. 정산 완료/미정산 토글 카운트가 갱신되고, 제목 수정 후 삭제된다. | `test.step("CP-08 ...")` |
| CP-09 | 참여 통계 | 생성한 스터디/루프팩 참여 데이터가 있는 상태에서 관리자 참여 통계와 팀별 참여율 화면이 열린다. 수정된 팀명이 통계 화면에 보인다. | `test.step("CP-09 ...")` |
| CP-10 | 모임 삭제 | 스터디와 루프팩 모임을 삭제하고 각 목록에서 상세 링크가 사라진다. | `test.step("CP-10 ...")` |
| CP-11 | 테스트 데이터 정리 | finally 단계에서 테스트 운영 단위를 삭제한다. 목록에서 사라지고 삭제된 입장 URL은 메인 경고 화면으로 이동한다. 삭제 실패는 삼키지 않고 테스트 실패로 처리한다. | `test.step("CP-11 ...")` |

## 실행 명령

```bash
set -a
source .env.local
set +a
RUN_CRITICAL_PATH_E2E=1 \
PLAYWRIGHT_BASE_URL="${PLAYWRIGHT_BASE_URL:-http://localhost:3000}" \
  npx playwright test --project=regression --retries=0 e2e/regression-critical-path.spec.ts
```

기본 회귀 실행에서는 skip 처리된다. 수동 실행 결과는 `CP-01`부터 `CP-11`까지 모두 PASS인지 확인하고, 실패하면 재시도 전에 실패한 `CP-*` 단계의 화면 상태를 먼저 조사한다.
