# UI system polish plan

## 목표

프로젝트 전반의 톤은 유지하되 운영툴로서 더 차분하고 읽기 쉬운 표면 체계를 만든다. 이번 루프는 큰 화면 재설계가 아니라 반복 사용되는 헤더, 카드, 리스트 밀도, 상태 칩을 정돈하는 1차 개선으로 제한한다.

## 변경 범위

| 파일 | 계획 |
|------|------|
| `src/app/globals.css` | 공통 표면 토큰을 다듬고, 카드/섹션/칩/툴바용 공통 클래스를 추가한다. 과한 hover 이동감은 줄이고 읽기 안정성을 높인다. |
| `src/app/dashboard-header.tsx` | 헤더를 더 낮고 정돈된 구조로 바꾸고, 기수 표시는 버튼처럼 보이지 않는 컨텍스트 라벨로 정리한다. 탭은 작고 명확한 세그먼트 스타일로 유지한다. |
| `src/app/meetup-dashboard.tsx` | 스터디/루프팩 카드의 정보 위계를 정돈한다. 시간, 비밀번호, 참여 수 같은 메타데이터는 공통 칩 스타일을 쓰고 본문은 스캔하기 쉽게 정리한다. |
| `src/app/afterparty/page.tsx` | 뒷풀이 카드도 스터디 카드와 같은 정보 위계와 카드 표면을 적용해 두 탭 간 시각적 차이를 줄인다. |

## 디자인 원칙

- 운영자가 반복해서 보는 화면이므로 마케팅형 장식보다 정보 밀도와 스캔성을 우선한다.
- 배경/카드/칩/버튼이 모두 같은 둥근 흰색 덩어리로 보이지 않게 역할별 표면을 나눈다.
- 기존 파란 accent와 밝은 회색 톤은 유지한다.
- 텍스트 크기는 뷰포트 기반으로 키우지 않고, 컴팩트한 계층만 조정한다.
- 새 dependency는 추가하지 않는다.

## 위험 요소

- 공통 CSS 변경은 여러 화면에 영향을 준다. 기존 클래스 이름은 유지하고 추가 클래스 중심으로 확장한다.
- 카드 hover/spacing 조정은 캡처 스크린샷에 영향을 줄 수 있다. 레이아웃이 밀리지 않도록 카드 내부 구조만 변경한다.
- 헤더 높이를 낮출 때 고정 헤더 spacer와 겹침이 생길 수 있다. 모바일/데스크톱 spacer를 함께 조정한다.
