# Member admin UX plan

## 목표

멤버 관리 팀 카드에서 보기와 편집 액션이 섞여 보이는 문제를 줄인다. 팀 카드 본문은 팀 구성 확인에 집중하고, 추가/삭제 액션은 명확한 버튼과 모달/행 액션으로 분리한다.

## 변경 범위

| 파일 | 계획 |
|------|------|
| `src/app/members/member-admin-form.tsx` | 팀 카드의 멤버 칩 UI를 리스트형으로 바꾸고, 멤버 추가를 모달로 이동한다. 삭제는 각 멤버 행의 `삭제` 버튼으로 표시한다. |

## UX 결정

- 팀 카드: 팀명, 엔젤 요약, 멤버 수, 주요 액션만 보여준다.
- 멤버 목록: 뱃지가 아니라 세로 리스트로 보여준다.
- 추가: 카드 안 인라인 입력 대신 `멤버 추가` 버튼 → 모달 입력으로 처리한다.
- 삭제: 작은 `×`가 아니라 명시적인 `삭제` 버튼으로 처리한다.
